Testando todas as GPIOs

Desenvolva um código para ESP32-Wroom-32 onde vamos acender e apagar o LED da placa usando a Arduino IDE para compilar.

Sim, é possível criar um código para testar todas as GPIOs da ESP32-WROOM-32 usando a Arduino IDE. A ESP32 possui até 34 GPIOs utilizáveis, mas nem todos são recomendados para uso geral devido a funções específicas (como entrada apenas, strap pins ou reservados para flash). Vou criar um código que testa os pinos mais comuns e seguros para uso como saída, piscando LEDs conectados a eles, e indicarei quais pinos evitar.

Considerações:

  • Pinos testáveis: Vou incluir os GPIOs mais comuns usados como saídas (ex.: 2, 4, 5, 12-19, 21-23, 25-27, 32-33). Pinos como 34-39 são apenas entrada e não podem ser usados como saída.
  • LEDs: Você precisará conectar LEDs (com resistores de 220Ω a 330Ω) entre cada GPIO testada e o GND.
  • Segurança: Evitarei pinos como GPIO 0, 1, 3, 6-11 (usados para flash) e outros com funções especiais.

Aqui está o código comentado em português:

// Array com os pinos GPIO a serem testados (saídas seguras)
int gpios[] = {2, 4, 5, 12, 13, 14, 15, 16, 17, 18, 19, 21, 22, 23, 25, 26, 27, 32, 33};
// Número de GPIOs no array
int numGpios = sizeof(gpios) / sizeof(gpios[0]);

void setup() {
  // Inicia a comunicação serial
  Serial.begin(115200);
  Serial.println("Teste de GPIOs da ESP32 iniciado!");

  // Configura todos os pinos como saída
  for (int i = 0; i < numGpios; i++) {
    pinMode(gpios[i], OUTPUT);
    digitalWrite(gpios[i], LOW); // Começa com todos desligados
  }
}

void loop() {
  // Testa cada GPIO, acendendo e apagando
  for (int i = 0; i < numGpios; i++) {
    Serial.print("Testando GPIO ");
    Serial.println(gpios[i]);
    
    digitalWrite(gpios[i], HIGH); // Acende o LED
    delay(500);                   // Espera 500ms
    digitalWrite(gpios[i], LOW);  // Apaga o LED
    delay(500);                   // Espera 500ms
  }
  
  Serial.println("Ciclo completo! Reiniciando...");
  delay(2000); // Pausa antes de reiniciar o ciclo
}

Como usar:

  1. Preparação:
    • Conecte LEDs aos pinos listados no array `gpios` (ex.: GPIO 2, 4, 5, etc.) com resistores em série (220Ω a 330Ω) até o GND.
    • Se não quiser testar todos, reduza o array `gpios` para os pinos que você tem LEDs conectados.
  2. Configuração na Arduino IDE:
    • Configure a IDE para a ESP32 (como explicado anteriormente: URL do gerenciador de placas e seleção de "ESP32 Dev Module").
    • Copie o código acima, compile e faça upload.
  3. Execução:
    • Abra o Monitor Serial (115200 baud) para ver qual GPIO está sendo testada.
    • Cada LED conectado piscará por 500ms, um de cada vez, em sequência.

Parte do DHT22

Ótimo! Fico feliz em saber que funcionou.

Aqui está um resumo dos passos que fizemos para instalar e fazer o sensor DHT22 funcionar com o ESP32:

  1. Instalação das Bibliotecas no Arduino IDE:
    • Abrimos o Gerenciador de Bibliotecas no Arduino IDE (Sketch > Incluir Biblioteca > Gerenciar Bibliotecas...).
    • Pesquisamos e instalamos a biblioteca "Adafruit Unified Sensor".
    • Pesquisamos e instalamos a biblioteca "DHT sensor library" (também da Adafruit).
    • Reiniciamos o Arduino IDE para que as bibliotecas fossem reconhecidas.
  2. Conexão Física do Sensor DHT22 ao ESP32:
    • Conectamos o pino DATA (ou OUT) do DHT22 ao GPIO 26 do ESP32 (que corresponde ao pino rotulado como D26 na maioria das placas).
    • Conectamos o pino VCC do DHT22 ao 3.3V ou 5V do ESP32.
    • Conectamos o pino GND do DHT22 a um pino GND do ESP32.
    • Verificamos a necessidade de um resistor pull-up (4.7kΩ a 10kΩ) entre o pino DATA e VCC do DHT22. (Módulos DHT geralmente já vêm com ele, mas sensores "nus" precisam de um externo).
  3. Verificação do Código e Teste:
    • Confirmamos que o código estava configurado para usar o pino correto (26) para o DHT22.
    • Realizamos testes (inclusive com o exemplo `DHTtester` da biblioteca) para garantir que a comunicação estava ocorrendo e que as leituras de temperatura e umidade eram válidas.
